# Leadping Python SDK

Typed Python client source for the Leadping API, generated from the public Leadping OpenAPI document with Microsoft Kiota.

This SDK gives Python applications a fluent, strongly typed way to call Leadping API resources such as leads, conversations, SMS, phone numbers, automations, reports, billing, and users.

## At a glance

| Item              | Value                        |
| ----------------- | ---------------------------- |
| Repository        | `leadpingai/leadping-python` |
| Package namespace | `leadping`                   |
| Main client       | `LeadpingOpenApiClient`      |
| Default API host  | `https://api.leadping.ai`    |
| OpenAPI source    | `leadpingai/openapi`         |
| Generator         | Microsoft Kiota `1.32.2`     |

## Status

This repository contains generated SDK source plus Python packaging metadata. The `Publish Python SDK` GitHub Actions workflow builds the wheel/source distribution, publishes to PyPI, tags the commit, and creates a GitHub release with the Python artifacts.

The generated source is refreshed from `leadpingai/openapi`. Do not hand-edit generated `.py` files; update the OpenAPI document and regenerate the SDK instead. This README is intended to be preserved across regenerations.

## Install

Install from PyPI after a version is published:

```bash
pip install leadping
```

The package includes the Kiota abstractions and serialization dependencies required by the generated client. Applications still choose their own Kiota request adapter and authentication provider. For the standard HTTP adapter used in the quick start:

```bash
pip install microsoft-kiota-http
```

## Quick start

```python
import asyncio
import os

from kiota_abstractions.authentication.authentication_provider import AuthenticationProvider
from kiota_http.httpx_request_adapter import HttpxRequestAdapter

from leadping import LeadpingOpenApiClient


class BearerTokenAuthenticationProvider(AuthenticationProvider):
    def __init__(self, token: str) -> None:
        self._token = token

    async def authenticate_request(self, request, additional_authentication_context=None) -> None:
        request.headers.add("Authorization", f"Bearer {self._token}")


async def main() -> None:
    token = os.environ["LEADPING_API_TOKEN"]
    adapter = HttpxRequestAdapter(BearerTokenAuthenticationProvider(token))
    adapter.base_url = "https://api.leadping.ai"

    client = LeadpingOpenApiClient(adapter)
    me = await client.users.me.get()

    print(me.email if me else None)


if __name__ == "__main__":
    asyncio.run(main())
```

For production applications, use an authentication provider that matches your Leadping credential flow and refreshes credentials as needed.

## Common entry points

The client exposes request builders for the main Leadping API areas:

| Area              | Builder                                                                           |
| ----------------- | --------------------------------------------------------------------------------- |
| Leads             | `client.leads`                                                                    |
| Conversations     | `client.conversations`                                                            |
| SMS               | `client.sms`                                                                      |
| Phone numbers     | `client.phone_numbers`                                                            |
| Automations       | `client.automations`                                                              |
| Sources           | `client.sources`                                                                  |
| Reports           | `client.reports`                                                                  |
| Users             | `client.users`                                                                    |
| Businesses        | `client.businesses`                                                               |
| Usage and billing | `client.usage`, `client.wallets`, `client.transactions`, `client.payment_methods` |

Each builder mirrors the OpenAPI path structure and provides typed request and response models from the `models` package.

## Regenerating

This SDK is regenerated by the Leadping GitHub workflow from the canonical OpenAPI repository:

```bash
kiota generate \
  --openapi ../openapi/openapi.json \
  --language Python \
  --output . \
  --class-name LeadpingOpenApiClient \
  --namespace-name leadping \
  --exclude-backward-compatible
```
